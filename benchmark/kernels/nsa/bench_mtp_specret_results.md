## 三、关键结论

| 维度 | 结论 |
|------|------|
| **纯 kernel** | 当 `B × d` 使 SM 饱和后，加速比线性逼近 `d`×。B=64/d=4/128K 达到 **3.993x**（理想 4x） |
| **端到端（无 CG）** | specret 有额外开销（repeat_interleave + sibling-patch + topk_transform），小 batch 下反而慢（<1x）；需要 B≥16~32（32K）或 B≥4~8（128K）才能盈利 |
| **端到端（有 CG）** | CUDA Graph 消除 CPU launch overhead 后，**所有场景均 ≥1x**（最低 1.017x @B=1/d=2/32K），小 batch 收益提升最大（从 0.65x → 1.3x） |
| **seq-len 影响** | 越长越早进入 compute-bound，相同 batch 下加速越高 |
| **CUDA Graph** | 是小 batch 场景下 specret 端到端有效的关键 enabler；大 batch 时 CG 与非 CG 差距收窄至 <5% |
| **推荐工作区** | 有 CG 时 d=4/B≥8 即可获得 **≥2x** 端到端加速（跨所有 seq-len） |
