Token 重排方法示意图

方法 A: Spectral CoOc (共选矩阵谱聚类)

1

收集所有 decode step 的 topk 选择

每步 indexer 选出 512 个 C4 压缩 token 位置。遍历全部 decode steps 记录每步的选中集合。

▼
2

构建共选矩阵 W (n×n sparse)

W[i,j] = token i 和 token j 在所有 steps 中被同一次 topk 同时选中的次数。W 是对称稀疏矩阵。

▼
3

归一化 Laplacian + 特征分解

L = I - D-1/2 W D-1/2 → 提取最小的 k 个特征向量 → 每个 token 得到一个 k 维谱嵌入。在谱空间里，经常共选的 token 距离近。

▼
4

重排: 同 cluster 的 token 连续放置

每个 cluster 对应一或多个连续 block。topk 选中的 token 更多落在同一个 block 内 → block scatter 降低。

为什么不能 online: 步骤 1 需要遍历所有 decode step 的 topk 选择来构建 W。Prefill 阶段还没有 decode，无法预知未来 query 会选中哪些 token pair。实验表明仅用 1 个 step 的 W 只减少 16% scatter（远不如 freq 的 50%）。

方法 B: Greedy CoOc (贪心共选 block-filling)

1

构建共选矩阵 W (同 Spectral)

遍历所有 decode steps，统计 token pair 共选次数 → 稀疏对称矩阵 W。

▼
2

逐 block 贪心填充

反复执行: 选频率最高的未分配 token 作为种子 → 用 sparse matvec 算所有候选与种子的共选亲和度 → top-k 一次填满整个 block。

与 Spectral 的区别: Spectral 用全局谱分解找最优 cluster，Greedy 逐块贪心填充直接最大化 block 内共选密度。Greedy 在 block 粒度指标上更优 (bs=64: scatter 5.0× vs spectral 7.2×)，但同样依赖完整 decode 历史的共选矩阵 W。
